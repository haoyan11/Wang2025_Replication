# 2. 今日修改总结 (2024-12-24)

## 🎯 核心工作（4个新文件）

### 2.1 新增Timing/Shape分解方法 ⭐⭐

**文件**: 03_decomposition_timing_shape.py (7.2KB)

**分解公式**:
```
TRc_y - TRc_av = ΔTR_timing + ΔTR_shape

其中:
- TRc_y   = Σ[t=SOS_y..POS_y] B_y(t)  (当年窗口内的实际TR)
- TRc_av  = Σ[t=SOSav..POSav] A(t)    (气候态窗口内的气候态TR)
- ΔTR_timing = Σ[t=SOS_y..POS_y] A(t) - TRc_av  (物候窗口变化贡献)
- ΔTR_shape  = Σ[t=SOS_y..POS_y] (B_y(t) - A(t)) (TR日变化贡献)
```

**进一步拆分**:
```
ΔTR_timing = ΔTR_SOS + ΔTR_POS
- ΔTR_SOS = Σ[t=SOS_y..POSav] A(t) - TRc_av  (SOS变化贡献)
- ΔTR_POS = Σ[t=SOS_y..POS_y] A(t) - Σ[t=SOS_y..POSav] A(t)  (POS变化贡献)
```

**输出**: 4个变量 × 37年 = 148个文件
- TRtiming_YYYY.tif
- TRshape_YYYY.tif
- TRsos_YYYY.tif
- TRpos_YYYY.tif

**与原方法对比**:
| 项目 | 原方法 (Wang 2025) | Timing/Shape方法 |
|------|-------------------|-----------------|
| 分解对象 | TRc = TRc_av + TRpheno + TRproduct | TRc - TRc_av = TRtiming + TRshape |
| 物候贡献 | TRpheno（窗口移动） | TRtiming（窗口时间变化） |
| 其他贡献 | TRproduct（其他） | TRshape（窗口内强度变化） |
| 进一步拆分 | ❌ | ✅ TRsos + TRpos |
| 输出变量 | 2个 | 4个 |

## 修正2: ΔSOS符号定义修正

**修正日期**: 2025-12-24
**影响文件**: 05_statistical_analysis.py, 05_statistical_analysis_T.py

### 问题确认

**原始错误** (Line 721):
```python
# ❌ 错误：使用了非标准的"提前量"定义
dsos_stack = sos_av - sos_stack  # dSOS = SOSav - SOS_year (advance > 0)
```

**问题分析**:
- 当SOS提前时，dSOS为**正值**（提前量）
- 不符合标准科学异常定义
- 导致回归斜率符号全部反转

### 修正方案

**新定义** (Line 721):
```python
# ✅ 正确：使用标准异常定义
delta_sos_stack = sos_stack - sos_av  # ΔSOS = SOS_year - SOSav (standard anomaly)
```

**标准异常定义**:
- SOS提前（变早）→ SOS_year < SOSav → **ΔSOS < 0**（负值）
- SOS推迟（变晚）→ SOS_year > SOSav → **ΔSOS > 0**（正值）

### 斜率符号变化

| 回归关系 | 原定义斜率 | 修正后斜率 | 物理解释 |
|---------|-----------|-----------|---------|
| TRc ~ ΔSOS | +1.515 | **-1.515** | SOS提前→TRc减少 ✓ |
| TRpheno ~ ΔSOS | +0.432 | **-0.432** | SOS提前→物候贡献减少 ✓ |
| TRproduct ~ ΔSOS | +1.139 | **-1.139** | SOS提前→生产力贡献减少 ✓ |

# 5. TRpheno符号关系说明

## 代码逻辑（03_decomposition_original.py）

```python
# Line 293-296: 确定符号
sign = np.where(
    sos_y_raw < sos_av_raw, +1.0,   # SOS提前 → sign = +1
    np.where(sos_y_raw > sos_av_raw, -1.0, 0.0)  # SOS推迟 → sign = -1
)

# Line 298-300: 计算区间和
start = np.where(sos_y_raw < sos_av_raw, sos_y_raw, sos_av_raw)
end = np.where(sos_y_raw < sos_av_raw, sos_av_raw, sos_y_raw)
sum_range = Σ[start:end] A(t)  # 总是正值

# Line 314: TRpheno = sign × sum_range
TRpheno = sign * sum_range
```

## 符号关系总结

| ΔSOS | 含义 | sign | TRpheno | 物理意义 |
|------|------|------|---------|---------|
| < 0 | SOS提前（sos_y < sos_av） | +1 | **正值** | 提前开始积累TR，增加贡献 ✓ |
| > 0 | SOS推迟（sos_y > sos_av） | -1 | **负值** | 延后开始，损失部分TR ✓ |
| = 0 | 无变化 | 0 | 0 | 无影响 |

## 重要澄清 ⚠️

**常见误解**:
- ❌ 错误：ΔSOS > 0（推迟）→ TRpheno > 0（正值）

**实际代码逻辑**:
- ✅ 正确：ΔSOS > 0（推迟）→ TRpheno < 0（**负值**）
- ✅ 正确：ΔSOS < 0（提前）→ TRpheno > 0（**正值**）

**物理意义**:
- SOS提前 → 生长季提前开始 → 在SOSav之前就开始积累TR → TRpheno为正（额外贡献）
- SOS推迟 → 生长季延后开始 → 错过了SOSav之前的一些TR积累 → TRpheno为负（损失）

这完全符合Wang 2025论文的物理解释。

## 6.1 固定窗口方法概述 ⭐⭐

### 背景与动机

**问题**：现有方法的局限性

| 方法 | 窗口定义 | 速率指标 | 窗口效应 | 问题 |
|------|---------|---------|---------|------|
| Wang2025 | 变化窗口 | TRproduct（累积） | ❌ 混入 | 无法区分窗口和强度效应 |
| Timing/Shape | 变化窗口 | TRshape（累积） | ❌ 混入 | 窗口效应掩盖真实速率 |
| 变化窗口Trate | 变化窗口 | TRc/GLS（归一化） | ⚠️ 掩盖 | 窗口归一化，接近0 |

**核心科学问题**：
> "春季提前年份，固定时间尺度下，蒸腾速率是否真的变高了？"

现有方法都无法直接回答，因为它们的速率指标都在**变化窗口**内计算，窗口效应掩盖了真实速率变化。

---

### 固定窗口方法（Method 2）核心思想

**分解公式**：
```
TRc_y = TRc_av + TR_window_change + TR_fixed_window

其中：
- TRc_av           = Σ[t=SOSav..POSav] A(t)         （固定窗口基线）
- TR_window_change = 窗口变化带来的TR累积            （SOS/POS效应）
- TR_fixed_window  = 固定窗口[SOSav,POSav]内的强度差异 （真实速率效应）⭐
```

**关键创新**：
```
Fixed_Trate = TR_fixed_window / (POSav - SOSav + 1)
```

**优势**：
- ✅ **固定尺度**：所有年份在相同窗口[SOSav, POSav]内比较
- ✅ **剥离窗口效应**：速率变化不受窗口延长/缩短影响
- ✅ **直接回答**："蒸腾速率是否真的变高变低？"

---

### 技术实现

由于没有当年每日TR数据，使用**增强因子假设**：

```python
# 1. 计算增强因子
enhancement_factor = TRc_y / sum_{SOS_y to POS_y} A(t)

# 2. 估算固定窗口内的当年累积
TRc_y_fixed = enhancement_factor × TRc_av

# 3. 固定窗口强度差异
TR_fixed_window = TRc_y_fixed - TRc_av

# 4. 窗口变化部分
TR_window_change = TRc_y - TRc_y_fixed

# 5. 固定窗口速率
Fixed_Trate = TR_fixed_window / Fixed_Window_Length
```

**假设**：窗口内的相对增强是空间均匀的（合理的一阶近似）

## 6.4 核心发现与解释

### 6.4.1 Fixed_Trate = -0.0128的含义

**斜率解读**：
```
Fixed_Trate ~ ΔSOS: 斜率 = -0.0128 mm/day per day ΔSOS

物理过程：
1. 春季提前年份（ΔSOS < 0）
   → 固定窗口[SOSav, POSav]内蒸腾速率增加
   → Fixed_Trate更高

2. 原因：春季提前年份整体偏暖
   → 植被更旺盛，光合作用更强
   → 同一时段内蒸腾速率提高
```

**与Wang2025对比**：

| 项目 | Wang2025预期 | 本研究结果 | 说明 |
|------|-------------|-----------|------|
| 速率变化方向 | 提前年份速率降低 | **提前年份速率增加** ❌ | 相反 |
| TRproduct符号 | 正值（补偿） | **负值（增强）** ❌ | 相反 |
| 物理机制 | 提前期气候偏冷 | **提前年份整体偏暖** | 数据源/区域差异 |

---

### 6.4.2 TRproduct符号问题的解释

**问题回顾**：
- Wang2025论文：TRproduct = +1.04 mm（正值，速率补偿效应）
- 本研究04a结果：TRproduct = -1.139 mm（**负值**，与论文相反）

**固定窗口方法的解释**：

```
TRproduct = -1.139（负值）
↓
Fixed_Trate = -0.0128（负斜率）
↓
含义：春季提前时，固定窗口内速率增加
↓
结论：TRproduct为负是合理的（增强效应，非补偿）
```

**原因分析**：
1. **数据源差异**：
   - Wang2025: GLEAM TR数据
   - 本研究: ERA5-Land TR数据

2. **区域差异**：
   - 北方森林春季提前年份可能整体偏暖
   - 温暖年份光合作用和蒸腾都增强

3. **时间段差异**：
   - 不同时期的气候背景可能不同

**科学意义**：
- ✅ **固定窗口方法成功解释了TRproduct符号**
- ⭐ **揭示了与Wang2025不同的机制**：窗口延长+速率增加（**双重增强**），而非窗口延长+速率降低（补偿）

---

### 6.4.3 三种速率指标对比

| 速率指标 | 窗口 | 斜率 | 物理含义 | 可靠性 |
|---------|------|------|---------|--------|
| **Trate** (变化窗口) | [SOS_y, POS_y] | +0.002 | 窗口归一化速率 | ⚠️ 掩盖 |
| **Fixed_Trate** (固定窗口) | [SOSav, POSav] | **-0.0128** | 固定尺度真实速率 | ✅ 可靠 |
| **TRshape/TRproduct** | [SOS_y, POS_y] | -1.004/-1.139 | 累积量（非速率） | ⚠️ 混入窗口 |

**结论**：
- ✅ **Fixed_Trate是唯一可靠的速率指标**
- ⚠️ Trate被窗口归一化掩盖（≈0）
- ⚠️ TRshape/TRproduct是累积量，混入窗口效应
